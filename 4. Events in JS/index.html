Although at this point we can now make almost any change we want to the DOM, from a user perspective it is not very helpful because we have only manually triggered changes. By learning about events, we will understand how to tie everything together to make interactive websites.
Events are actions taking place at the browser that are initiated by the user or the browser itself. Some common examples inclde ;

  • User clicking a button
  • User hovering a mouse.
  • User submitting a form.
  • User pressing a key on the keyboard.
  • A page finishes loading.
By writing JS sripts that response to these events, developers can customise messages to be displayed, valdate information, react and many more. Lets cover event listeners, event handlers, and event objects as this will give you more manouverability in developing interactive web experiences for end users.

1.2 Event Handlers and Listeners.

Developers can create triggers that fire an event(action) when userc click a button or press a key. Such events are commonly reffered to as keypress or click events. An event handler is basically a JavaScript function that executes when an event fires. While an event listener attaches a responsive interface to an element. It allows a particilar element to wait and listen for a event event to fire.
There are 3 ways in which we can assign events events to elements.

1.3. Common Events.

We have learned about inline event handlers, event handler properties, and event listeners using the click event, but there are many more events in JavaScript.

(a) Mouse Events.

These are among the most frequently used events. They refers to those events that involve clicking buttons on the mouse or hovering and moving the mouse pointer.

    Event - Description
  • click - Fires when mouse is pressed and released on an element
  • dblclick - Fires when an element is clicked twice
  • mouseenter - Fires when a pointer enters an element
  • mouseleave - Fires when a pointer leaves an element
  • mousemove - Fires when a pointer moves inside an element
Using mouseenter and mouseleave in tandem recreates a hover effect that lasts as long as a mouse pointer is on the element.

(b) Form Events.

Form events are actions that pertain to forms, such as input elements being selected or unselected, and forms being submitted. JavaScript is often used to submit forms and send the values through to a backend language. The advantage of using JavaScript to send forms is that it does not require a page reload to submit the form, and JavaScript can be used to validate required input fields

(c) Keyboard Events.

They are used for handlng Keyboard activities such as pressing a key, lifting a key, and holding down a key.
